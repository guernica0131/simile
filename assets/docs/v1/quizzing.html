Creating your quiz

To create your quiz, send a POST to the following URL

POST https://projectsimile.org/api/v1/quizzes

The expected response is as follows.

{
    "name": "Simile Sample Quiz",
    "timed": "0",
    "random": false,
    "retakes": 1,
    "published": false,
    "ordered": false,
    "createdAt": "2014-05-29T09:15:52.409Z",
    "updatedAt": "2014-05-29T09:15:52.409Z",
    "id": 5
}

A quiz object is comprised of the following paramters

  • Name describing your quiz

    name :: string , required

  • Lessons, this quiz can be mapped to specific lessons built into Simile. The array must be composed of the IDs of valid lesson objects

    lessons :: array of integers , optional

  • Questions, zero or manu question assined to this quiz's question bank. This value is not required, but a valid quiz session cannot be started until at least one question has been assined. The array must be composed of the IDs of valid quiz objects

    questions :: array of integers , optional

  • Time, sets the time restriction in minutes for the quiz. If 0, then there will be no restriction

    timed :: integer , defaults to 0

  • Random, this value will randomize the question ordering for each session.

    random :: boolean , defaults to false

  • Minumum, defines a minumum number questions that should appear in the quesion bank for each session. 

    min :: integer , optional

  • Maximum, sets the maximum number of questions in the question bank. If no value is provided, all questionbank questions for the quiz will appear in the session. Note: if both min and max are set and max > min, a random inclusive sample will be taken for a quiz session. For example, suppose we have min = 5 and max = 10, a random number of questions between 5 and 10 will be assined to a session.

    max :: integer , optional

  • Retakes, defines the number of times a user can take a quiz. If this value is set to 0, then there are no restrictions. Set to n where n is an integer greater than 0 to have n chance[s] to take a quiz.

    retakes :: integer , defaults to 0

  • Published, if a quiz is publsihed, it will be accessable to other users for taking. Otherwise, only administrators will be allowed access.

    published :: boolean , defaults to false

  • Ordered, if set to true, this parameter ensure that the user answers all questions in succession. Please note, if true, paging to one question at a time is enforced until the system marks the prior question complete.

    ordered :: boolean , defaults to false

Questions can be assined to zero or many quiz objects in Simile. You can assign existing questions at anytime to a

Starting your quiz

POST https://projectsimile.org/api/v1/quiz/:id/start

If we make a valid request, we would see the following response

{"session": 2} // 2 is the session id

An invalid request would have to form

{"error": "Error message"}

Reasons for an invalid request:

  • In session error: 
    {error: "You already have a quiz in session. Please resume your quiz with session id "}
    The user is attempting to take a quiz that is already in session. If the users completes the quiz or his/her time expires (in the case that it is a timed quiz), then the user will be allowed to start another session for the number or retakes defined for that quiz.
  • Max retakes error: 
    {error:  "You've exceeded the maximum number of retakes for the quiz. Please contact your quiz adminstrator for more information"}
    The user has reach the allowed number of retakes. To prevent this, you should set your quiz parameter, "retakes" to 0 for unlimited retakes.
  • Invalid parameters error: 
    {error:  "Insufficient parameters to initiate this quiz"}
    The POST requestion must include a valid quiz id as shown above.
  • Unkown error: 
    {error:  "There was an error processing this quiz request. Please try again."}
    There was a system error that prevented the processing of your request. To report this issue, please post to our issues tracker.
  • Publication error: 
    {error:  "You are attempting to take an unpublished quiz. Please contact your quiz administrator for further instructions"}
    The quiz has not been published. This can be resolved by setting the quiz parameter "published" to true.
  • Inactive error: 
    {error:  "This quiz is currently inactive. Contact your quiz adminstrator for further instructions."}
    This error will occur when no questions have been assinged. To correct this error assing one or many questions and ensure your quiz has been published.
  • Ivalid error: 
    {error:  "You are attempting to take an invalid quiz. Please contact your quiz adminstrator."}
    This error occurs when there is a mismatch with your quiz parameter. The most likely case is an error in the quiz paramters.
  • Undefined error: 
    {error:  "There is no quiz matching this ID"}
    There is no quiz for matching the posted id.
    • Min/Max mismatch error: 
    {error: "The minimum number of questions assigned to this exceeds either the maximum parameter or number of questions in the question bank. Please contact your quiz adminstrator for more details"}, make sure your quizzes min attribute is less than or equal to the max and the number of questions in the question bank

Getting the questions

Once we have a valid session id, we can call retrieve our questions with the following GET request

GET https://projectsimile.org/api/v1/quizSessions/:id?{params}

If no parameters are defined, Simile sends back all questions in the bank. However, we can target specif questions assingd to our quiz session with the following parameters

  • Returns a specific question 
    api/v1/quizSessions/:id?question:3 // 3 is the third question in                                  // the question bank. Not the                                   // question model's id.
  • Returns your questions in paged form. You must supply a valid end parameter. If your end exceeds the question bank size, all values will be return. If a start is not supplied, Simile will return all values to the specified end. The 'question' parameter will return a spcific question
    api/v1/quizSessions/:id?paged=true&start=0&end=4
api/v1/quizSessions/:id?question=3
    These values are interpreted as array indices. Therefore, following the above example, index 0 is the first question while 4 is the 5th question. This value is inclusive, so all questions between and equal to 0 - 5 will be returned.

Note: If the 'ordered' parameter is set to true, Simile will ignore all paging requests and only return the question and question state currently assigned to the session
